Coastal Innovation Hackathon

How TARANG coordinates a rescue from first distress ping to final closure.

TARANG is built for coastal operators who need fast visibility, simple decisions, and a direct response flow. Every active case moves through one operational track: detect, verify, dispatch, monitor, and resolve.

Operations View
Alert intake

Distress reports enter through SOS events, operator actions, and tracked profiles.

Case focus

Selecting a profile should take the operator straight to the live map and case actions.

Process
Step 01
Signal captured

The system receives a distress report and marks the case active inside the operations center.

Step 02
Operator review

The operator opens the profile, confirms the incident, and sees the case location on the map.

Step 03
Rescue dispatch

Dispatch actions assign the nearest team and move the case into a tracked response state.

Step 04
Live monitoring

Vessel movement, case status, and operator alerts stay visible until the rescue is complete.

What the operator needs
Clarity
  • Only active cases should dominate the dashboard.
  • Resolved cases should stay visible in their own summary state.
  • The control center must close cleanly and return to the case list.
Speed
  • Profile selection should immediately focus the map on that exact case.
  • Actions should stay inside the operations flow, not hide in dead-end screens.
  • Dismissible messages should never trap the operator for a fixed timeout.
Communication
  • Language switching should stay consistent across text, alerts, and voice.
  • Only one completion voice should play after rescue resolution.
  • Operator feedback sounds must support action instead of adding noise.
TARANG is designed to feel like an operations console, not a demo page. The goal is fast action with minimal confusion.
